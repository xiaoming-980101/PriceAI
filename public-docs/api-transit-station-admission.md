# API 中转站收录与合作说明

PriceAI 做的是 API 中转站的公开比价、资料核验和可用性观察。我们不售卖 API，也不替任何商家做担保。收录和合作的前提是：站点愿意把用户购买前必须知道的价格、倍率、模型分组、上游类型和可用性信息公开出来。

## 收录状态

| 状态 | 含义 |
| --- | --- |
| 正式上架 | 满足公开资料、分组披露、价格口径和监测页要求，可以进入站点榜和详情页 |
| 待核验 | 基础资料存在，但部分分组、上游来源、监测样本或价格口径还不完整 |
| 仅后台线索 | 只有站点线索或私聊资料，暂不对外展示 |
| 暂不收录 | 缺少公开价格和监测信息，或拒绝披露关键来源类型 |

正式上架不等于 PriceAI 推荐。榜单排序仍以价格、可用性、样本数、资料完整度和用户反馈为主；商业合作会单独标识，不直接影响客观排序。

## 正式上架的基础门槛

正式上架至少需要满足以下条件：

1. 提供可访问的官网地址。
2. 官网必须有公开价格页或公开只读数据接口。
3. 公开充值倍率，例如 `1 元 = 1 额度`、`1 元 = 1.2 额度`。
4. 公开最低充值、余额有效期、退款规则和售后入口。
5. 公开所有对外展示的模型分组倍率。
6. 每个模型分组必须说明上游来源、账号池或渠道类型。
7. 每个要展示的模型分组必须有可用性监测页或可被 PriceAI 读取的监测数据。
8. 每个模型或模型分组需要有可对照的模型基准价。
9. 公开基础限流口径，例如并发数、RPM、TPM、单 Key 限制或账号级限制。

API Base URL 不是强制提交项。PriceAI 当前更关注用户购买前能否公开核验价格、倍率、分组来源和可用性；如后续做真实调用监测，再单独向站点申请专用测试 Key 或测试账号。

## 模型分组披露要求

站点不能只写“高速池”“稳定池”“高级池”这类模糊名称。每个模型分组都需要说明它实际代表什么。

建议至少公开：

| 字段 | 说明 |
| --- | --- |
| 分组名称 | 站内展示的分组名 |
| 覆盖模型 | 该分组可用的模型列表或标准模型映射 |
| 分组倍率 | 该分组相对基准价或站内计费的倍率 |
| 模型基准价 | 用于计算倍率的公开基准价来源 |
| 上游类型 | 官方 API、云厂商、订阅转 API、逆向、二级分销、混池等 |
| 账号池类型 | Free、Plus、Pro、Max、Team、Enterprise、官方 API、混池等 |
| 可用性监测 | 该分组对应的公开监测页或监测数据入口 |
| 限流口径 | 并发数、RPM、TPM、单 Key 限制、账号级限制或是否未承诺 |
| 更新时间 | 价格、倍率和监测数据最后更新时间 |

## 合作与展示边界

- PriceAI 可以展示商家提供的公开资料、优惠码、测试结论和用户反馈摘要。
- 商业合作、赞助展示或 AFF 关系必须单独标识。
- 合作关系不能直接改变客观排序、最低价计算和风险提示。
- 站点资料发生变化时，应及时同步公开价格页、监测页或公开快照接口。
