# 架构说明

PriceAI 是一个 Next.js + Supabase 的 AI 订阅报价聚合工具。核心思路是把“来源渠道里的原始商品”整理成“用户能比较的标准商品”。

## 数据流

```text
来源渠道
  -> 采集器 / 导入器
  -> raw_offers 原始报价
  -> 分类规则 / 人工覆盖
  -> canonical_products 标准商品
  -> 前台比价 / 详情 / 全部报价视图
```

## 核心表

| 表 | 作用 |
| --- | --- |
| `sources` | 来源平台或卡网，包含名称、入口链接、采集方式和启用状态 |
| `raw_offers` | 原始商品事实层，保留标题、价格、库存、购买链接、来源和采集时间 |
| `canonical_products` | 标准商品，例如 `ChatGPT Plus`、`Gemini Pro` |
| `offer_matches` | 原始报价到标准商品的匹配关系，可自动生成或人工修正 |
| `crawl_runs` | 每次采集结果、成功数、失败原因和日志摘要 |

## 前台

前台围绕用户购买前决策设计：

- 首页展示标准商品聚合结果。
- 最低价只来自有效有货报价。
- 详情页展示某个标准商品下的全部渠道报价。
- 全部报价视图展示某个平台下的所有原始报价。
- 缺货报价保留可见，但不参与最低价。

## 后台

后台围绕“让采集更稳定”设计：

- 管理来源渠道。
- 单渠道试采集和批量采集。
- 查看采集日志和失败原因。
- 审核用户提交渠道。
- 调整商品分类和报价显示状态。

人工补录只作为调试或临时兜底，不是长期维护价格的主路径。
