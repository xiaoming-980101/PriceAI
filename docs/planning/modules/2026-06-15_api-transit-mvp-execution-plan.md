# PriceAI API 中转站 MVP 执行规划

> 文档类型：PRD Lite + 执行清单
> 日期：2026-06-15
> 分支：`codex/api-transit-mvp`
> 状态：执行中
> 目标：把现有 `/api-transit` 草稿收敛为“价格 + 稳定性优先”的可验证 MVP

## 1. 产品一句话

API 中转站 MVP 是 PriceAI 面向中转 API 用户的价格与稳定性比较工具，帮助用户在 Claude / GPT 中转站之间快速比较充值系数、模型倍率、综合倍率、可用性和轻量风险提示，从而判断哪些站点值得小额试用。

## 2. 版本边界

### 本版本做什么

1. 以“渠道榜 / 站点榜”为主入口，模型榜作为辅助视图。
2. 模型范围先收敛到两类：`Claude` 和 `GPT`。
3. 统一展示标准模型名：`Claude Sonnet 4.6`、`Claude Sonnet 4.7`、`Claude Sonnet 4.8`、`GPT 5.5`、`GPT 5.4`。
4. 按站点分别计算 `Claude 最低综合倍率` 和 `GPT 最低综合倍率`。
5. 支持三种排序：综合、倍率、稳定性。
6. 数据先使用静态配置，不接自动采集；所有样例数据明确标注状态。
7. 补齐 `/api-transit/submit` 和 `/guides/api-transit`，避免主路径断链。
8. 保留轻量风险提示，但不做复杂检测报告、评分或仲裁机制。

### 本版本不做什么

1. 不直接售卖 API、充值、发卡或托管余额。
2. 不让普通用户提交生产 API Key 做检测。
3. 不做自动采集、实时监控、任务 Bench、TTFT/TPS/并发检测。
4. 不做商家后台、订单、保证金、返佣结算。
5. 不把入驻或返佣包装成客观排名，不承诺担保。

## 3. 目标用户与场景

| 用户 | 场景 | 触发点 | 期望结果 | MVP 支持 |
| --- | --- | --- | --- | --- |
| API 散户用户 | 想给 Claude Code、Cursor、Codex 或自建工具找中转站 | 官方 API 成本或支付门槛高 | 找到便宜且相对稳定的站点 | 站点榜、Claude/GPT 综合倍率、可用性、风险提示 |
| 有经验的中转用户 | 在多个站点之间换备用 Key | 某站涨价、不可用、号池不稳 | 快速比较哪个站更适合小额试用 | 综合/倍率/稳定性排序，详情价格拆解 |
| 中转站站长 | 希望被收录或申请入驻 | 需要曝光和可信资料页 | 知道该提交哪些资料 | `/api-transit/submit` 分角色提交说明 |
| PriceAI 运营者 | 维护第一批样例站点 | 不想一开始接自动采集 | 能通过静态配置先跑通页面 | typed local data + 明确字段契约 |

## 4. 核心数据口径

### 4.1 价格公式

```text
充值系数 = 站内额度换算比例 / 7.2
综合倍率 = 充值系数 * 模型倍率
```

说明：

- `1:1` 表示 1 元人民币约等于 1 站内美元额度，充值系数约 `0.14x`。
- Claude 和 GPT 分别计算最低综合倍率，不混成一个全站最低值。
- 如果站点只公开模型倍率但没有充值比例，则综合倍率显示为 `未公开`。

### 4.2 列表页核心列

| 列 | 用途 |
| --- | --- |
| 站点 | 名称、官网、数据状态、商业关系 |
| 号池来源 | Pro / Plus / Max / Team / 官方 API / 混池 / 未披露 |
| 充值系数 | 展示换算后的系数和原始比例 |
| Claude | Claude 倍率区间 + Claude 最低综合倍率 |
| GPT | GPT 倍率区间 + GPT 最低综合倍率 |
| 近 7 日稳定性 | 站点整体可用率、样本数、最后检查时间 |
| 风险提示 | 样本不足、二级分销、混池、上游未披露、反馈待核验等 |
| 更新时间 | 最近人工确认时间 |

### 4.3 排序策略

| 排序 | 目标 | 规则 |
| --- | --- | --- |
| 综合 | 默认排序，兼顾价格与稳定性 | 优先低综合倍率，同时给高可用性、样本数、资料完整度、入驻资料加轻权重 |
| 倍率 | 单纯找低价 | 按 Claude / GPT 中可用的最低综合倍率升序 |
| 稳定性 | 找更稳的站 | 按近 7 日可用率、样本数、最近确认时间排序 |

## 5. 页面结构

### 5.1 `/api-transit`

目标：用户在 30 秒内筛出 2-3 个值得小额试用的站点。

必须包含：

- 标题、边界说明：不售卖 API，不担保，建议小额试用。
- 指标卡：已收录站点、Claude 最低综合倍率、GPT 最低综合倍率、近 7 日样本数。
- 搜索、模型、渠道类型、账号池筛选。
- 三种排序：综合、倍率、稳定性。
- 桌面表格与移动紧凑列表。

### 5.2 `/api-transit/[slug]`

目标：用户判断单个站点是否值得小额试用。

必须包含：

- 站点基础信息、官网、商业关系、数据状态。
- 充值规则、最低充值、余额有效期、售后渠道。
- Claude 价格表和 GPT 价格表。
- 站点整体稳定性与模型分组稳定性。
- 轻量风险提示和用户反馈摘要。

### 5.3 `/api-transit/models`

目标：保留模型维度辅助比较，但不抢主入口。

必须包含：

- 只展示 Claude / GPT 两类模型。
- 按标准模型名汇总各站点报价。
- 展开后展示站点、模型倍率、综合倍率、稳定性、风险提示。

### 5.4 `/api-transit/submit`

目标：承接普通用户推荐、站长入驻、上游/批发商线索。

MVP 做静态说明，不做数据库写入。

必须包含：

- 三类提交路径：用户推荐、商家入驻、上游/批发。
- 每类需要准备的字段。
- 明确不要提交 API Key、账号密码、敏感密钥。
- 联系方式先走 Telegram / GitHub / 站内反馈。

### 5.5 `/guides/api-transit`

目标：解释中转站名词和比较方法。

必须包含：

- 中转站是什么。
- 倍率和真实成本怎么算。
- 号池来源、混池、二级分销、未披露的含义。
- 为什么 PriceAI 先展示价格和稳定性，检测放 V2。
- 小额试用和风险控制建议。

## 6. 实现计划

1. 更新静态数据模型和 helper，保留 typed local data。
2. 改造站点榜：字段、筛选、排序、移动端卡片。
3. 改造详情页：价格拆解和稳定性优先。
4. 简化模型页：只保留 Claude / GPT。
5. 补齐顶部导航、提交页、指南页、sitemap/guide directory 入口。
6. 跑 `npm run lint`、`tsc --noEmit`、`npm run build`。

## 7. 验收标准

- `/api-transit` 默认显示渠道榜，并按综合排序。
- Claude / GPT 最低综合倍率分别展示，不混合。
- 模型筛选只出现 `Claude`、`GPT`。
- 页面不再把 Gemini、DeepSeek、Codex 作为 MVP 核心模型。
- `/api-transit/submit` 和 `/guides/api-transit` 不再 404。
- 顶部导航出现 `中转 API` 或等价入口。
- 构建输出包含 `/api-transit`、`/api-transit/models`、`/api-transit/submit`、`/guides/api-transit`。
- `npm run lint`、`tsc --noEmit`、`npm run build` 通过。

## 8. 后续 V2 候选

- 接入真实检测结果：真模型、协议兼容、扣量异常、TTFT/TPS。
- 商家资料提交入库和后台审核。
- 用户反馈/投诉证据流。
- 入驻、优选、返佣、保证金的商业披露与治理规则。
- 站点可用性定时检测和历史曲线。
